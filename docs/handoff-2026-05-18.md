# Session Handoff

> Created: 2026-05-18
> AlphaLog(구 Trading Diary) UI·브랜딩·데이터 모델 정리 및 CSS 레이어 이슈 수정.

## Context

- 앱 공식명 **AlphaLog**. 로고: `public/images/logo.png` (`/images/logo.png`).
- 로그인(Auth): 타이틀 AlphaLog, 서브 **「나만의 초과수익(α) 매매 일지」**, 상단 `<img>` 로고 72px 플레이스홀더.
- 사이드바(`BottomNav.tsx`) 동일 브랜딩·로고, 하단 `AlphaLog v1.0`.
- **`index.css`**: 전역 `* { margin: 0 }` 등을 **`@layer base`** 안으로 옮김 → Tailwind `mb-*` / `mt-*` / `mt-auto`가 정상 적용되던 문제 해결(언레이어드 규칙이 utilities를 덮어쓰던 현상).
- **매매 기록(`NewTrade`)**: 상단 카드에 1주당 가격·수량·총액 자동 계산; 기준 선택은 체크만. 저장 시 금액은 **가장 작은 option_number 한 행**에만, 나머지 선택은 `price=0, quantity=0` 태그 행.
- **DB**: `supabase/migrations/003_trade_options_tag_rows.sql` — `trade_options.quantity >= 0` 허용. **미적용 시 Supabase에 반드시 적용**할 것.
- **프로젝트 메타**: `package.json` name `alphalog`, `index.html` title AlphaLog, `vite.config.ts` PWA name/short_name AlphaLog.
- `README.md`·`package-lock.json` 등 일부 경로에 구 이름 잔존 가능 → 필요 시 정리.
