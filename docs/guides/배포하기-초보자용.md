# AlphaLog를 인터넷에 올려서 다른 사람도 쓰게 하기 (초보자용)

이 문서는 **처음 배포해 보는 분**도 위에서부터 순서대로 따라 하면 되도록 적었습니다.  
한 번에 다 하지 않아도 됩니다. **중간에 멈췄다가** 나중에 이어서 하면 됩니다.

---

## 0. 먼저 이해하기 (1분)

- **AlphaLog 화면**은 웹사이트 파일이라 **Vercel** 같은 곳에 올립니다. (무료로 가능)
- **로그인·데이터베이스**는 **Supabase**가 맡습니다. (무료로 가능)
- 여러 사람이 **같은 주소**로 들어와 **각자 회원가입**하면, 데이터는 **계정마다 따로** 저장됩니다. (이미 코드에 그렇게 되어 있습니다.)

**준비물**

| 필요한 것 | 용도 |
|-----------|------|
| [GitHub](https://github.com) 계정 | 코드를 인터넷 저장소에 올리기 |
| [Supabase](https://supabase.com) 계정 | 로그인·DB |
| [Vercel](https://vercel.com) 계정 | 웹 주소 제공·자동 빌드 |

모두 무료 플랜으로 시작할 수 있습니다.

---

## 1단계: 코드를 GitHub에 올리기

> 이미 이 프로젝트가 GitHub에 있다면 **2단계**로 넘어가세요.

### 1-1. Git이 설치돼 있는지 확인

PowerShell 또는 터미널에서:

```bash
git --version
```

버전이 나오면 OK입니다. 안 나오면 [Git 공식 사이트](https://git-scm.com/download/win)에서 Windows용을 설치한 뒤 터미널을 다시 엽니다.

### 1-2. GitHub에서 새 저장소 만들기

1. [github.com](https://github.com)에 로그인합니다.
2. 오른쪽 위 **+** → **New repository**를 누릅니다.
3. **Repository name**에 예: `alphalog` 처럼 이름을 적습니다.
4. **Public**으로 두어도 됩니다. (.env 같은 비밀 파일은 아래 규칙대로 안 올라갑니다.)
5. **Create repository**를 누릅니다.

### 1-3. 내 PC 폴더에서 Git 연결 후 올리기

PowerShell에서 **프로젝트 폴더로 이동**합니다. 폴더 이름에 공백이 있으므로 따옴표를 씁니다.

```powershell
cd "c:\dev\Trading Diary"
```

아직 Git을 안 썼다면:

```powershell
git init
git add .
git commit -m "AlphaLog 초기 업로드"
```

GitHub 새 저장소 페이지에 나오는 주소를 복사합니다. (HTTPS 형태 예: `https://github.com/내계정이름/alphalog.git`)

```powershell
git branch -M main
git remote add origin https://github.com/내계정이름/alphalog.git
git push -u origin main
```

처음이라 GitHub 로그인 창이 뜰 수 있습니다. 안내에 따라 진행하면 됩니다.

### 1-4. 비밀이 올라가지 않았는지 확인 (.env)

이 프로젝트의 `.gitignore`에 이미 `.env`가 포함되어 있습니다.  
**반드시** Supabase 키·비밀번호는 `.env`에만 두고, GitHub에는 **커밋하지 마세요.**

## 2단계: Supabase에서 프로젝트와 DB 준비

### 2-1. Supabase 회원가입·새 프로젝트

1. [supabase.com](https://supabase.com)에서 가입 또는 로그인합니다.
2. **New project**를 누릅니다.
3. 이름·데이터베이스 비밀번호·지역을 정하고 만들기까지 **1~2분** 걸릴 수 있습니다.

### 2-2. SQL로 테이블 만들기 (마이그레이션 순서 중요)

1. 왼쪽 메뉴에서 **SQL Editor**를 클릭합니다.
2. **New query**를 눕니다.
3. PC에서 아래 세 파일을 **메모장이나 에디터로 연 뒤, 전체를 복사**해서 Supabase SQL 창에 붙여 넣습니다.  
   **항상 순서대로**, **한 번에 하나씩** 실행하는 편이 안전합니다.

| 순서 | 파일 위치 |
|-----|-----------|
| 1번 | `supabase/migrations/001_initial_schema.sql` |
| 2번 | `supabase/migrations/002_unlimited_options.sql` |
| 3번 | `supabase/migrations/003_trade_options_tag_rows.sql` |

각각 붙여 넣은 뒤 우측 하단 **Run** (또는 실행)을 누릅니다.  
에러 없이 성공했다는 메시지가 나오면 다음 파일로 넘어갑니다.

이미 예전에 1번만 실행한 적이 있다면, **안 한 것만** 실행하면 됩니다.

### 2-3. API 주소와 anon 키 복사해 두기

1. 왼쪽 **Project Settings**(톱니바퀴) → **Data API** 또는 **API** 메뉴로 갑니다.
2. 아래 두 가지를 메모장에 복사해 둡니다. (나중에 Vercel에 넣습니다.)

- **Project URL** → 나중에 환경 변수 이름: `VITE_SUPABASE_URL`
- **anon public** 키 (긴 문자열) → 나중에: `VITE_SUPABASE_ANON_KEY`

> `service_role` 키는 **절대** 웹이나 GitHub에 넣지 마세요. 사용하지 않아도 됩니다.

### 2-4. 인증 URL은 나중에 다시 옵니다

지금은 Vercel 주소를 아직 모를 수 있으므로, **4단계**에서 다시 설정합니다.

---

## 3단계: Vercel에 연결해서 사이트 만들기

### 3-1. Vercel 가입

1. [vercel.com](https://vercel.com)에서 **Sign up**합니다.
2. **Continue with GitHub**를 선택하면 연동이 쉽습니다.

### 3-2. GitHub 저장소 가져오기

1. Vercel 대시보드에서 **Add New…** → **Project**를 누릅니다.
2. **Import** 옆에 본인 GitHub의 `alphalog`(또는 올린 저장소 이름)를 고릅니다.
3. **Import**를 누릅니다.

### 3-3. 빌드 설정 확인

보통 자동으로 맞춰집니다. 아니면 다음으로 맞춥니다.

- **Framework Preset**: Vite
- **Build Command**: `npm run build`
- **Output Directory**: `dist`
- **Install Command**: `npm install`

### 3-4. 환경 변수 입력 (매우 중요)

같은 화면의 **Environment Variables**에 다음을 추가합니다. (이름 철자 그대로!)

| Name | Value |
|------|--------|
| `VITE_SUPABASE_URL` | 2-3에서 복사한 Project URL |
| `VITE_SUPABASE_ANON_KEY` | 2-3에서 복사한 anon public 키 |

경제지표(FRED)를 쓸 계획이면 선택으로:

| `VITE_FRED_API_KEY` | FRED 발급 키 |

**Environment**: Production에 체크되어 있는지 확인합니다. (Preview도 같이 쓰려면 동일 변수를 추가해도 됩니다.)

### 3-5. 배포하기

**Deploy**를 누릅니다. 1~3분 정도 지나면 **Visit** 같은 버튼과 함께 주소가 나옵니다.  
예: `https://alphalog-xxx.vercel.app`

이 주소를 메모장에 **그대로** 복사해 둡니다.

---

## 4단계: Supabase에 “실제 웹 주소” 등록하기

배포가 끝났으면 Supabase가 “이 주소에서 오는 로그인만 허용”하도록 맞춥니다.

1. Supabase 프로젝트로 돌아갑니다.
2. 왼쪽 **Authentication** → **URL Configuration**으로 갑니다.
3. **Site URL**에 Vercel 주소를 넣습니다. 예: `https://alphalog-xxx.vercel.app`
4. **Redirect URLs**에 아래를 **한 줄씩 추가**합니다.  
   - `https://alphalog-xxx.vercel.app`  
   - `https://alphalog-xxx.vercel.app/**`  
   (본인 주소로 바꾸세요.)
5. 로컬에서 개발할 때도 쓰려면 같은 목록에 예를 들어 다음도 추가합니다.  
   - `http://localhost:5173`
6. **Save**를 누릅니다.

## 자주 하는 질문 (문제 해결)

**Q. 화면은 뜨는데 로그인이 안 돼요.**  
- Supabase의 **Site URL**, **Redirect URLs**에 Vercel 주소가 정확히 들어갔는지 다시 확인하세요. (`https`, 끝에 `/` 차이까지 주의)

**Q. 배포했는데 Supabase 변경이 안 먹는 것 같아요.**  
- Vercel에서 **Deployments** 탭으로 가서 최신 배포의 **⋯**에서 **Redeploy** 해보세요. 환경 변수 수정 후에는 다시 빌드가 필요합니다.

**Q. 새로고침하면 페이지가 깨져요.**  
- 이 저장소에는 Vercel용 `vercel.json`이 있습니다. 그래도 안 되면 Vercel 프로젝트 설정의 Output Directory가 `dist`인지 확인하세요.

**Q. 다른 사람에게 어떻게 알려줘야 해요?**  
- Vercel 주소 링크만 보내면 됩니다. 들어간 사람은 **직접 회원가입**하면 각자 계정으로 사용합니다.
